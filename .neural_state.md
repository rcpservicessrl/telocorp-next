2. **Deploy:** `cd c:\dev\telocorp-next && echo y| npx vercel deploy --prod --yes`. CLI falla con archivos >50MB — usar GitHub para assets grandes.

3. **Imágenes productos:** Las 71 imágenes de catálogo están en `public/TeloCorp/images/` (Vercel las sirve estáticamente). URLs en Supabase = `https://telocg.com/TeloCorp/images/imageX.png`. El proxy CDN `wsrv.nl` las optimiza a WebP.

   **PLAN DE IMÁGENES (resuelto):**
   - Origen: recuperadas del git history del proyecto legacy (commit `7f9a0c5` en `c:\dev\telocorp`)
   - Copiadas a: `public/TeloCorp/images/` en el proyecto Next.js
   - URLs en Supabase: `https://telocg.com/TeloCorp/images/imageX.png`
   - Servidas por: Vercel (archivos estáticos en `public/`)
   - Optimización: `wsrv.nl/?url={URL}&w=400&output=webp&q=75`
   - **Nuevas imágenes:** se suben desde admin via ImgBB (componente `ImageUpload`). La URL resultante (https://i.ibb.co/...) se guarda en el producto.
   - **NO borrar `public/TeloCorp/` del repo** — si se borra, las imágenes legacy dejan de servirse
   - **ImgBB key:** `8199e433dfe9c12d1f452ce857dbce9d` (free tier, solo upload)

4. **Supabase Auth SSR:** `@supabase/ssr` en browser y server. `window.location.href` (no router.push) para que el middleware detecte cookies.

5. **Gemini API key AQ.:** Formato permanente. Si da 429, la API "Generative Language" no estaba habilitada en Google Cloud Console. Ir a console.cloud.google.com, habilitarla, regenerar la key.

6. **WhatsApp:** +1 (809) 903-8707 — en chat fallback, footer, notificaciones.

7. **Resend:** `noreply@telocg.com` como from. Templates en `lib/email.ts`. Se activa con `RESEND_API_KEY` en env vars.

8. **Cantidad de pasos en tsc:** Siempre correr `npx tsc --noEmit` antes de deploy. Si hay errores de tipos, resolver antes.

9. **Git workflow:**
   ```
   git add -A
   git commit -m "feat:descripcion-corta"
   git push origin master
   echo y| npx vercel deploy --prod --yes
   ```
